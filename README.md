# 📚 TXT → EPUB 웹 리더 서비스

> 다양한 형식의 장문 TXT 파일을 EPUB 전자책으로 변환하고,
> 
> 
> 웹에서 개인 전자책을 열람할 수 있도록 만든 개인용 웹 서비스입니다.
> 

---

## 🔍 프로젝트 개요

장문의 텍스트 문서를 웹에서 읽기 불편하다고 느껴,

TXT 파일을 전자책 형식으로 변환하고 **웹 기반 EPUB 뷰어**로 열람할 수 있도록 제작한 프로젝트입니다.

- TXT 파일 업로드
- 자동 EPUB 변환
- 웹 기반 전자책 뷰어 제공
- 개인 계정 기반 라이브러리 관리

---

## ✨ 주요 기능

### 1. TXT → EPUB 자동 변환

- 다양한 챕터 형식 자동 인식
    
    (`n화`, `[제목] n화`, `n화. 부제`, `n화：부제` 등)
    
- 인식 실패 시 5000자 단위 자동 분할 fallback
- UTF-8 / CP949 인코딩 자동 처리
- HTML escape 적용으로 안전한 렌더링

### 2. 웹 기반 EPUB 리더

- 페이지 단위 이동
- 키보드 / 마우스 조작
- 챕터 기반 내비게이션
- 검색 및 검색 결과 하이라이트
- 읽기 위치 자동 저장 (CFI 기반 북마크)
- 스타일 커스텀 조작(폰트 크기, 글자간격, 줄간격, 테마)

### 3. 개인 라이브러리 관리

- 사용자별 파일 분리 저장
- 업로드한 전자책 목록 조회
- 책 삭제 및 북마크 관리

## 🛠 기술 스택

### Frontend

- **Next.js (App Router)**
- **TypeScript**
- **epub.js / react-reader**
- Tailwind CSS
- shadcn/ui

### Backend

- **Django**
- **Django REST Framework**
- JWT 기반 인증
- ebooklib (EPUB 생성)

### Storage

- Django FileField 기반 파일 저장
- EPUB 파일만 저장 (원본 TXT 미보관)

```

        ↓

        ↓

        ↓
[웹 EPUB 뷰어로 렌더링]
        ↓
[CFI 기반 북마크 저장]

```

---

## 🧠 설계 포인트

### 1. 실패하지 않는 변환 로직

- 챕터 인식 실패 시에도 EPUB 생성 보장
- 변환 실패로 인한 사용자 경험 저하 방지

### 2. EPUB 표준 준수

- spine / toc / nav / ncx 구성
- 다양한 전자책 뷰어 호환성 고려

### 3. 보안 및 데이터 분리

- 인증된 사용자만 접근 가능
- 사용자별 파일 완전 분리
- 서버에 원본 TXT 내용 저장하지 않음

---

## 🔐 권한 및 보안

- 모든 API는 인증 사용자만 접근 가능
- 사용자는 본인의 책만 조회 / 삭제 가능
- 파일 공유 및 공개 기능 없음

---

## 🧪 예외 처리

- 잘못된 인코딩 파일 처리
- 챕터 패턴 미존재 시 자동 분할
- 변환 실패 시 예외 반환
